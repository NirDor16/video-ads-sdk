Multi-Screen Applications

The SDK fully supports applications with multiple screens (multiple Activity instances).

You only need to initialize the SDK once. No additional integration is required in other screens.

Where to Initialize the SDK

Call AdsSdk.init() a single time during app startup. The recommended locations are:

  • Preferred: Application.onCreate()
  • Alternative: MainActivity.onCreate()
AdsSdk.init(
  context = applicationContext,
  baseUrl = "https://video-ads-sdk.onrender.com/",
  appId = "demo_app"
)

How It Works

After initialization, the SDK automatically:

  • Tracks all Activity lifecycle events globally
  • Monitors user interactions (clicks) across all screens
  • Detects the currently active screen
  • Displays ads on the active screen when a trigger condition is met

This is implemented internally using Application.ActivityLifecycleCallbacks and window-level touch interception.

Behavior Across Screens

User interactions are counted continuously across screens. The click counter is not reset when navigating between activities.

When an ad is triggered:

  • The ad is displayed on the currently visible screen
  • The previous screen state is preserved
  • After closing the ad, the user returns to the same screen

Important Notes

  • No SDK code is required in secondary activities
  • The SDK automatically ignores the ad player screen itself
  • Only one ad can be displayed at a time

In summary: multi-screen support is fully automatic and requires no additional developer effort beyond a single initialization call.
